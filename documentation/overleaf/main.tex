\begin{abstract}
This document describes the implementation of a 2D virtual space where users move in real time and automatically connect to chat when they are within a proximity radius. The system is built with a React + PixiJS client and a Node.js + Socket.IO server. It includes organized activity rooms, room-based tools, direct proximity chat, typing indicators, mini-map, and user profile customization.
\end{abstract}

\section{Project Overview}
The goal of this project is to simulate real-world proximity interactions in a 2D environment. Users can move using the keyboard, see other participants, and automatically connect to a direct chat when they are close. Additional room-based features enable themed spaces with activity-specific tools.

\section{Tech Stack}
\begin{itemize}[leftmargin=*]
  \item \textbf{Frontend}: React, Vite, PixiJS, Socket.IO Client
  \item \textbf{Backend}: Node.js, Express, Socket.IO
  \item \textbf{Persistence}: In-memory state (session-based, no DB)
\end{itemize}

\section{Core Features}
\begin{itemize}[leftmargin=*]
  \item Real-time movement (WASD / Arrow keys)
  \item Visible multi-user avatars in a shared canvas
  \item Proximity detection and automatic connect/disconnect
  \item Direct chat only when in range
  \item Clean UI with world view and chat panel
\end{itemize}

\section{Extra Features Implemented}
\begin{itemize}[leftmargin=*]
  \item Organized activity rooms with fixed layout
  \item Mini-map with room outlines and user dots
  \item Room Directory with live occupancy
  \item People list with status and direct chat shortcuts
  \item Profile customization (name, color, status, hat, glasses)
  \item Emotes (wave/heart/clap) rendered above avatars
  \item Room chat mode separate from direct chat
  \item Typing indicators for both chat modes
  \item Room tools: shared timer and shared sticky notes
  \item Presence history (recently left)
\end{itemize}

\section{System Architecture}
\subsection{High-Level Flow}
\begin{enumerate}[leftmargin=*]
  \item Client connects via Socket.IO
  \item Server assigns user to an activity room and spawn point
  \item Client sends movement updates
  \item Server broadcasts position updates and manages proximity
  \item When distance \textless radius, a direct chat room is created
  \item When distance \ge radius, the direct chat room is left
\end{enumerate}

\subsection{Client Responsibilities}
\begin{itemize}[leftmargin=*]
  \item Render the world (PixiJS)
  \item Handle keyboard input and local movement
  \item Emit movement updates
  \item Show proximity ring and room areas
  \item Render UI panels (rooms, people, chat, profile)
  \item Manage chat modes and typing state
\end{itemize}

\subsection{Server Responsibilities}
\begin{itemize}[leftmargin=*]
  \item Track connected users and positions
  \item Perform proximity detection
  \item Manage direct chat rooms
  \item Manage room chat channels
  \item Maintain room tools (timer, notes)
  \item Broadcast user updates and presence
\end{itemize}

\section{Real-Time Events}
\subsection{Client to Server}
\begin{longtable}{p{0.32\linewidth}p{0.6\linewidth}}

\textbf{Event} & \textbf{Purpose} \\
\midrule

room:message & Room chat message \\
direct:typing & Direct chat typing indicator \\

\bottomrule
\end{longtable}

\section{Proximity Logic}
\begin{itemize}[leftmargin=*]
  \item Each user has a circular proximity radius.
  \item If distance between two users is less than the radius, they are paired into a direct chat room.
  \item If distance is greater than or equal to the radius, the room is left and chat is disabled.
\end{itemize}

\section{Room Organization}
Rooms are rectangular zones with predefined positions. New users spawn in the least populated room at a fixed grid point to keep the layout organized. Each room has an associated activity (e.g., Focus, Sprint, Lounge).

\section{UI/UX Summary}
\begin{itemize}[leftmargin=*]

  \item Mini-map provides quick navigation.
  \item Right panel: Rooms, People, Profile, Chat.
  \item Chat supports Room and Direct modes, with typing indicators.
\end{itemize}

\section{Limitations}
\begin{itemize}[leftmargin=*]
  \item In-memory state only (no persistence across restarts)
  \item No audio or recording features
  \item Basic avatar rendering (no uploaded assets)
